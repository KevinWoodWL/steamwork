# 蒸汽系统

Steamwork 的蒸汽系统参考机械动力风格的锅炉：供水、热源、锅炉规模和机器负载共同决定有效输出。
实现上保持 Pylon/Rebar 原生风格，用流体和机器状态表达蒸汽，不直接照搬机械动力的应力单位模型。

## 核心概念

- 热量：锅炉被加热的强度。
- 供水：每 tick 或每周期进入锅炉的水量。
- 锅炉容量：安全压力容器的规模或等级。
- 压力：内部储存的蒸汽压力。
- 蒸汽输出：提供给管道和机器的流体输出。

## 核心公式

```text
steamOutput = min(boilerCapacityLevel, heatLevel, waterInputLevel) * efficiency
```

产汽量高于消耗量时压力上升，机器消耗蒸汽时压力下降。

## 热源等级

1. 被动热源：营火/熔岩相邻一类的低效热源。
2. 燃烧热源：煤或木炭火箱，作为前期标准输出。
3. 过热热源：生物柴油或高级燃烧器，作为高阶输出。

## 锅炉等级

### 铜锅炉

- 前期单方块机器。
- 通过 Rebar 流体输入接收 Pylon 的水。
- 通过 Rebar 流体输出产出 Steamwork 的蒸汽。
- 第一版使用下方方块热源：营火、火、熔岩、岩浆块。
- 适合作为第一批蒸汽锤、小型泵或蒸汽物流的动力来源。

### 蒸汽动力臂

- 参考机械动力的动力臂：在半径约 5 的立体范围内，在多个输入/输出点之间搬运物品。
- Steamwork 版本使用蒸汽压力，而不是 SU/RPM。
- 当前实现会扫描附近原版库存和 Pylon/Rebar 物流组，从可输出组取物并插入可输入组。
- 后续应支持玩家显式标记输入/输出目标，选择轮询或优先级模式，并通过资源包显示机械臂路径。

### 青铜锅炉

- 中期机器或小型多方块结构。
- 需要 Pylon 青铜板和更好的流体处理。
- 支持稳定的中压运行。

### 钢制高压锅炉

- 后期多方块结构，预计由 MonolithLib 支持。
- 需要 Pylon 钢板、耐火材料和流体舱口。
- 支持高压和多个蒸汽消费者。

## 蒸汽等级

Steamwork 有两种蒸汽流体，由不同锅炉产出，给不同档次的机器使用：

| 流体 | 产生方 | 消费方 |
|---|---|---|
| `steam` | 青铜锅炉、因瓦锅炉 | 现有所有蒸汽机器 |
| `superheated_steam` | 锰钢锅炉、钨锅炉 | 高阶机器（卷管机、结晶器、蒸馏塔、超压加压炉，待实装） |

升级到锰钢/钨级锅炉是一次"换轨道"：失去对旧机器的兼容，换来对新一档机器的解锁。
青铜/因瓦锅炉本身没有被淘汰，前中期机器仍然由它们驱动。

## 安全机制

- 低压：待机或输出不足。
- 中压：高效工作区间。
- 高压：警告状态。
- 超压：泄出蒸汽、损伤内部进度，后续可考虑受控爆炸。

第一版应优先泄压，而不是爆炸。等玩家有足够清晰的 UI 反馈后，再考虑爆炸机制。

## 资源包状态键

```text
steamwork:copper_boiler:active=true
steamwork:copper_boiler:pressure=low
steamwork:copper_boiler:pressure=medium
steamwork:copper_boiler:pressure=high
steamwork:copper_boiler:pressure=venting
steamwork:steam_arm:active=true
```

这些字符串应视为资源包公开接口。

## 蒸汽传输

蒸汽作为 Rebar 流体，直接通过 Pylon 提供的流体管道传输，Steamwork 不再
提供独立的 `steam_pipe` 或 `pressure_valve` 方块，避免和 Pylon 管道系统冲突。
