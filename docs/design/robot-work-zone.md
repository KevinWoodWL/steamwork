# 机器人工区（Robot Work Zone）设计方案

> 目标版本：v0.5.0 / P4 重构。
> 把"单体机器人 + GUI 切模式"重构成**控制终端管理的工区系统**：终端=大脑，区域=工作范围，
> 出货点/充汽点=绑定点，四种单职责机器人=工人。

## 一、最终形态

- **控制终端（Robot Control Terminal）**：新机器方块，带 GUI。统管一个工区：
  - **区域**：矩形（两角 A/B，<b>忽略 Y</b> 的竖直棱柱），上限 32×32（settings 可调）。用<b>区域设定器</b>点端点 1/端点 2 划定。
  - **出货点**：区域内一个容器。采矿/砍树机器人把库存满了送来。
  - **充汽点**：区域内一台蒸汽充汽舱。机器人缺汽来此充。
  - **机器人槽位**：GUI 里若干槽，放入机器人物品即<b>部署</b>对应机器人（移除即召回）；终端统管其归属。
  - 约束：出货点、充汽点必须落在区域内。
- **四种机器人**（独立合成、单职责，不再 GUI 切模式）：
  - 采矿机器人：区域内挖原版矿石 → 7 格库存 → 满了送出货点。
  - 砍树机器人：区域内砍原版原木 → 7 格库存 → 满了送出货点。
  - 搬运机器人：① 出货点 → 产线入口/外部容器；② 区域内绑定的两容器互运。两种皆可。
  - 巡逻机器人：区域内巡游（观赏/占位；后续可挂警戒等）。
- **机器人行为**（共性）：只在<b>所属终端的区域内</b>搜目标作业；7 格内置库存自收掉落；缺汽自动去充汽点；
  破坏走保护事件、跳过未加载区块、绝不破坏 Rebar 方块（沿用现有安全约束）。

## 二、与现状的关系（重构）

会<b>取代</b>现在 SteamRobot 的这些：per-robot `home`/`work-radius`、GUI 切 jobMode、HAUL 自带源/目标绑定、
自动绑定充汽舱。改为：机器人<b>读所属终端</b>的区域/出货/充汽配置；job 由机器人<b>类型</b>固定。

保留复用：寻路与站位（findStandingSpot）、清挡路树叶、破坏进度/裂纹、整树/矿脉 flood-fill、
brain 清空与交互 guard、活动状态 WAILA、开箱盖/取放动画、区块安全守卫。

## 三、数据模型

- `RobotControlTerminal`（RebarBlock + Gui）：
  - `cornerA, cornerB`：BlockVector（x,z；Y 忽略），持久化；`region` 由两角推出。
  - `deliveryPoint, chargePoint`：Location，持久化；置点时校验在区域内。
  - `robotInventory`：VirtualInventory（槽数 = 最大机器人数，默认如 4）；放入机器人物品 → 部署。
  - 静态注册表 `ACTIVE_TERMINALS: Map<UUID, RobotControlTerminal>`（仿 ProductionLineInlet.ACTIVE_INLETS），
    供机器人按 terminalId O(1) 找回终端。
- `SteamRobot`（RebarEntity）：
  - 新增 `terminalId`（绑定终端，持久化）+ `RobotType`（MINE/CHOP/HAUL/PATROL，部署时定，持久化）。
  - 去掉 home/workRadius/jobMode 切换；区域/出货/充汽改从 `terminal()` 读。
  - 7 格库存（`carrying`，byte[] 持久化）。

## 四、区域设定器（Region Wand）

- 一个 RebarItem。流程：右键<b>控制终端</b> → 绑定为"当前配置的终端"（记在 wand PDC）；
  右键方块 → 设端点 1；再右键 → 设端点 2 → 写入该终端，校验 ≤32×32、刷新 GUI。
- 备选（更省）：终端 GUI "设定区域"按钮 → SteamworkBlockPrompt 连点两次设角。实现时择一。

## 五、分阶段路线

| 阶段 | 内容 | 产出 |
|---|---|---|
| **A 终端地基** | 终端方块 + 状态（区域/出货/充汽）+ 持久化 + 注册表 + 区域设定器划区 + 置点校验 | 能放置、能划区、能置点的终端（无机器人） |
| **B 机器人改造** | SteamRobot 加 terminalId + RobotType；去 home/模式切换，改读终端；4 种机器人物品；终端槽位部署/召回 | 终端槽位放物品→机器人在区域内动 |
| **C 库存+出货** | 7 格库存、自收掉落、满了送出货点（容器/产线入口） | 采矿/砍树闭环 |
| **D 充汽点** | 缺汽寻路去充汽点充满返回（替换自动绑定舱） | 自维持 |
| **E 搬运+巡逻+收尾** | 搬运两模式、巡逻；4 种机器人合成/研究/指南页 | 功能完整、可正常获取 |

## 六、待定 / 风险

- **多终端/区域重叠**：机器人按 terminalId 归属（槽位部署即绑定），不靠"在谁区域内"猜，避免重叠歧义。
- **终端破坏**：召回所有所属机器人（掉回物品 + 库存）。
- **机器人越界**：只在区域内选目标；若被推出区域，寻路回区域内。
- **性能**：区域最大 32×32×worldHeight 扫描很大——分块/限频扫描，沿用 isChunkLoaded 守卫与扫描冷却。
- **充汽点**：要求是蒸汽充汽舱（复用现有充能逻辑），还是任意"充汽标记点"？倾向前者（复用）。
