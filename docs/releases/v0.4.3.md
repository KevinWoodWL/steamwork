新增
--

*   **阶段 2 汽动逻辑层**：新增 5 台机器，构成「蒸汽即信号」的完整闭环——从蒸汽升压、逻辑门控、节拍整形到模拟红石输出，再到物品网络阀控
*   **蒸汽涡旋管**（SteamVortexTube）：将过热蒸汽分离为加压蒸汽（南面输出）和蒸馏水（东面输出），约 10% 转换损耗；是「过热→加压」链路的唯一途径，蒸馏水可用于下游化学配方
*   **汽动逻辑门**（PneumaticLogicGate）：AND / OR / XOR / NOT 四种门；以相邻机器蒸汽量为布尔信号（≥阈值=HIGH），逻辑成立时把真实蒸汽搬运到输出面相邻机器；对三种蒸汽档位各自独立运算
*   **蒸汽振荡器**（SteamOscillator）：从输入面持续蓄压，达到上阈值后向输出面喷发节拍蒸汽 N tick，压降至下阈值后重新蓄压；供汽越快频率越高，无蒸汽则停摆
*   **汽动截止阀**（PneumaticGateValve）：嵌入汽动导管链的双端口直通节点；无红石信号时常开，接收红石信号后立即截断汽动物品网络
*   **蒸汽压力传感器**（SteamPressureTransducer）：读取相邻机器指定蒸汽档位的填充比例，通过桶库存映射为原版比较器可直接读取的 0–15 模拟红石信号；是蒸汽体系唯一的真模拟量红石输出

变更
--

*   **汽动截止阀纳入汽动研究**：截止阀加入「精密汽动物流」研究的解锁物品列表，与导管、输入端、输出端、货运站同组解锁
*   **新增汽动逻辑研究**：涡旋管、逻辑门、振荡器、压力传感器归入新研究「精密汽动逻辑」，与精密汽动研究并列
*   **指南页面更新**：截止阀加入「汽动物流」子页面；涡旋管、逻辑门、振荡器、压力传感器加入「蒸汽自动化」子页面
*   **三档蒸汽 GUI 统一**：振荡器和压力传感器均采用 `SteamLogicSupport.SteamKind` 单一档位选择器，普通 / 过热 / 加压独立显示，Waila 压力条颜色跟随所选档位
*   **研究重命名**：原「精密蒸汽自动化」（含蒸汽臂、装配台、蒸汽马达）拆分为「精密装配」和「精密高级自动化」，装配台单独成研究，蒸汽臂与马达归入高级自动化

修复
--

*   **汽动端点容器识别**：汽动输入端、输出端在导管平行放置时（导管不与容器同轴），原来会把导管反面误判为容器方向，导致物品推送/抽取失效；现在改用独立于导管位置的 `containerAccessFace()` 检测，平行导管不再影响容器识别
*   **蒸汽振荡器喷发受阻状态**：输出端满载或缺失时 GUI 和 Waila 显示红色「喷发受阻」提示，帮助玩家排查管线断连
*   **压力传感器桶库存修正**：比较器占位物使用专用 PersistentData 标记；若被漏斗或玩家扰动，下一 tick 自动恢复到目标红石等级
*   **汽动导管刷新重入保护**：导管 `refreshDisplays` 加入重入锁，避免邻接通知风暴触发递归更新
